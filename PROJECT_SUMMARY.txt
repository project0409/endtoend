### Daily Usage:
```bash
# Terminal 1: MongoDB
mongod

# Terminal 2-5: All Backends
cd Module1_UserAuthentication\backend && npm start
cd Module2_ExperienceSharing\backend && npm start
cd Module3_PreparationGuidance\backend && npm start
cd Module4_SkillGapAnalysis\backend && npm start

# Terminal 6: Integrated Frontend
cd IntegratedApp\frontend && npm start
```

**Access:** http://localhost:3000

- JWT Authentication

- MongoDB 4.4+
- Database: interview_prep_platform
- Collections: users, experiences, resources, checklists, assessments

## 🔐 AUTHENTICATION FLOW

1. User registers (Module 1)
2. JWT token generated
3. Token stored in localStorage
4. Token sent with all API requests
5. All modules use same token
6. Seamless navigation between modules

## 📡 API ENDPOINTS

### Module 1 (Port 5001)
- POST /api/auth/register
- POST /api/auth/login
- GET /api/auth/me
- PUT /api/auth/profile

### Module 2 (Port 5002)
- POST /api/experiences
- GET /api/experiences
- GET /api/experiences/:id
- GET /api/experiences/my-experiences
- PUT /api/experiences/:id
- DELETE /api/experiences/:id

### Module 3 (Port 5003)
- POST /api/resources
- GET /api/resources
- POST /api/checklists
- GET /api/checklists
- GET /api/checklists/:id
- POST /api/checklists/:id/toggle

### Module 4 (Port 5004)
- POST /api/assessments
- GET /api/assessments
- GET /api/assessments/:id
- PUT /api/assessments/:id
- DELETE /api/assessments/:id
